# Majed — Reply-to-message: ما يُضاف إلى برومبت بوت بريس

عندما يضغط العميل على زر «رد» في الودجت ويختار رسالة سابقة، يقوم الجسر (`integration/bridge/index.js`)
بتضمين الرسالة المقتبسة **داخل نص رسالة العميل** قبل إرسالها إلى بوت بريس (لأن الـ Chat API لا
يملك حقلاً مستقلاً للاقتباس). تصل الرسالة للبوت بهذا الشكل الثابت:

```
↩️ ردًّا على رسالة ماجد السابقة: «نص الرسالة المقتبسة»
سؤال العميل أو ردّه هنا
```

أو إذا كان العميل يردّ على رسالته هو:

```
↩️ ردًّا على رسالة العميل السابقة: «نص الرسالة المقتبسة»
سؤال العميل أو ردّه هنا
```

## النص المطلوب إضافته في System Prompt / Instructions الخاصة بماجد

انسخ هذا المقطع داخل تعليمات البوت:

```
## التعامل مع الردود المقتبسة

أحيانًا تبدأ رسالة العميل بسطر اقتباس بهذا الشكل:
↩️ ردًّا على رسالة ماجد السابقة: «نص الرسالة المقتبسة»
أو:
↩️ ردًّا على رسالة العميل السابقة: «نص الرسالة المقتبسة»

هذا السطر ليس سؤال العميل، بل يوضّح الرسالة التي يردّ عليها.
- اعتبر النص داخل «...» هو السياق الذي يشير إليه العميل.
- افهم باقي الرسالة (السطر الذي يلي الاقتباس) على أنه ردّ العميل أو سؤاله المرتبط بهذا السياق.
- لا تُعِد كتابة سطر الاقتباس في ردّك، وردّ مباشرةً على قصد العميل.
```

## الصيغة (Format) — حافظ على التطابق

الصيغة مُولّدة من الجسر هنا:

- `integration/bridge/index.js` → معالج `POST /widget/message` (يبني سطر `↩️ ردًّا على …`).
- `integration/bridge/public/majed-widget.js` → `startReply` / `postMsg` (تلتقط الاقتباس وترسله).

العلامات المميِّزة التي يعتمد عليها البوت:
- يبدأ السطر دائمًا بـ `↩️ ردًّا على`.
- النص المقتبس دائمًا بين «...» (مقصوص إلى 240 حرفًا كحد أقصى).
- يليه سطر جديد يحوي رسالة العميل الفعلية.

> لو غيّرت هذه الصيغة في الكود، حدّث هذا الملف وبرومبت بوت بريس معًا ليظلّا متطابقين.
